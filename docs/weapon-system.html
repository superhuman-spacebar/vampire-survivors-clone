무기 시스템 리팩토링

Vampire Survivors Clone — 무기를 레고 블록처럼 조합해서 만드는 시스템으로 전환

지금은 어떻게 되어 있나?

현재 게임에는 무기가 3개 있다. 채찍(Whip), 미사일(Magic Missile), 성수(Holy Water).

문제는 이 3개가 전혀 다른 방식으로 만들어져 있다는 것이다.

🗡 채찍

앞쪽을 휘두른다. 적 하나하나와의 거리를 직접 계산해서 범위 안에 있으면 때린다.

⚡ 미사일

총알이 날아간다. 게임 엔진이 총알과 적의 충돌을 알아서 감지한다.

💧 성수

바닥에 물웅덩이를 깐다. 시계처럼 일정 간격으로 범위 안 적에게 데미지를 준다.

예를 들어 "모든 무기에 적을 느리게 만드는 효과를 넣고 싶다"고 하면, 채찍 코드에 둔화 로직을 넣고, 미사일 코드에도 넣고, 성수 코드에도 따로 넣어야 한다. 3개면 3번, 10개면 10번. 방식이 다르니까 복사도 안 되고 각각 다르게 만들어야 한다.

뭐가 문제인가?

문제 1: "적을 느리게 만드는 총알"을 만들 수 없다.

둔화 기능을 총알 코드 안에 직접 만들어야 한다. 다른 무기에 같은 효과를 넣고 싶으면? 코드를 통째로 복사해야 한다.

"둔화" 효과를 넣고 싶은 무기를 클릭해 보세요.

문제 3: "적에게 데미지를 준다"는 같은 행동인데, 무기마다 처리 과정이 다르다.

채찍, 미사일, 성수 모두 결국 "적에게 데미지를 준다". 그런데 그 과정이 무기마다 완전히 다르게 만들어져 있다. 그래서 "모든 무기에 넉백을 넣자"고 하면 무기 하나하나의 과정을 전부 따로 수정해야 한다.

무기를 선택해서 각각 데미지를 주는 과정이 얼마나 다른지 비교해 보세요.

문제 4: 적이 나를 때리는 것도 같은 시스템이어야 한다.

"데미지를 준다"는 행위는 플레이어 → 적, 적 → 플레이어 모두에서 일어난다. 지금은 적이 플레이어를 때리는 것은 완전히 별도의 코드다. 만약 적에게 "접촉 시 둔화"나 "접촉 시 화상" 효과를 넣고 싶다면? 또 따로 만들어야 한다. 플레이어가 쓰는 것과 적이 쓰는 것이 같은 시스템이면, 효과를 한번 만들고 양쪽에서 재사용할 수 있다.

어떻게 바꿀 것인가?

핵심 아이디어는 간단하다. 무기를 하나하나 처음부터 만드는 대신, 4가지 블록을 조합해서 무기를 찍어내는 것이다.

무기 = 🎯 타겟 + ⚡ 전달 + ❄ 효과 + 🔥 이벤트

이 4가지 블록을 골라서 끼우면 무기가 완성된다. 햄버거를 만들 때 빵, 패티, 소스, 토핑을 고르는 것과 같다.

Block 1: 🎯 누구를 공격할 것인가? (Targeting)

첫 번째 블록은 타겟을 정하는 방식이다. 5가지 중 하나를 고른다.

가장 가까운 적

제일 가까운 적을 조준

보는 방향

캐릭터 이동 방향

랜덤 위치

근처 아무 곳에

범위 전체

주변 모든 적

회전

내 주위를 빙글빙글

Block 2: ⚡ 어떻게 데미지를 줄 것인가? (Delivery)

두 번째 블록은 데미지를 전달하는 방식이다.

🔨 발사체

총알이 날아감

🗡 근접

앞쪽을 즉시 휘두름

💧 장판

바닥에 반복 데미지

🔄 회전

내 주위를 돌며 데미지

💫 오라

항상 주변에 데미지 영역

기존 무기 3개도 이 블록으로 표현할 수 있다:

  • 채찍 = 보는 방향 + 근접
  • 미사일 = 가장 가까운 적 + 발사체
  • 성수 = 랜덤 위치 + 장판

Block 3: ❄ 맞은 적에게 무슨 일이 일어나는가? (Effects)

세 번째 블록은 상태 이상 효과다. 핵심은 "얼음 총알"이라는 별도의 무기를 만드는 게 아니라, 아무 무기에든 효과를 붙이기만 하면 된다는 것이다.

❄ 둔화 — 속도 50% 감소, 2초간
🔥 화상 — 지속 데미지, 3초간
🥶 빙결 — 완전 정지, 1초간
💥 넉백 — 밀쳐내기

효과는 "무슨 종류, 얼마나 강하게, 몇 초 동안"이라는 숫자 설정으로만 정의된다. 코드가 아니라 데이터다. 여러 효과를 동시에 붙일 수도 있다. 예를 들어 "화상 + 넉백" 총알도 가능하다.

Block 4: 🔥 특별한 순간에 무슨 일을 하는가? (Hooks)

네 번째 블록은 이벤트다. 무기가 동작하는 중 특정 순간에 원하는 행동을 연결할 수 있다.

🚀 생성될 때

총알이 발사되는 순간

💥 적에게 맞을 때

총알이 3개로 분열!

💀 적을 처치할 때

체력 1 회복

💣 소멸할 때

사라지면서 폭발!

이 이벤트 안에서 또 다른 무기 효과가 발동될 수 있다. 이것이 연쇄 효과의 핵심이다.

🔨 발사체가 날아간다 → 💥 적에게 맞는다 → 💣 3개의 작은 총알로 분열! → ❄ 각 총알이 둔화 효과 적용
💧 장판을 깐다 → ⏰ 지속시간이 끝난다 → 💣 소멸 이벤트 발동! → 💥 폭발 + 넉백

총알에서 총알이 나오고, 장판이 끝나면서 폭발하고, 그 폭발에서 또 다른 효과가 나올 수 있다. 이벤트를 이벤트에 연결하는 것이다.

데미지 처리의 통합

지금은 채찍, 미사일, 성수가 각각 다른 경로로 데미지를 처리한다. 그래서 효과를 넣으려면 3군데를 다 고쳐야 한다.

새 시스템에서는 어떤 무기든 같은 하나의 경로로 데미지를 처리한다.

❌ 지금: 무기마다 다른 길

채찍: 거리 계산 → 데미지 → 처치 알림

미사일: 충돌 감지 → 데이터 읽기 → 데미지 → 별도 처리

성수: 타이머 → 거리 계산 → 데미지 → 처치 알림

⚠ 효과를 하나에 넣으면 나머지에는 안 됨

✅ 이후: 모든 무기가 같은 길

→ 데미지 적용

→ 효과 자동 적용 auto

→ 넉백 자동 적용 auto

→ "맞았을 때" 이벤트 발동

→ 처치? → "처치" 이벤트 + 경험치

✅ 한번 만들면 전 무기에 자동 적용

비교: "얼음 발사체"를 추가한다면?

❌ 지금

1. 새 무기 동작 코드 작성 (~80줄)

발사체 생성, 조준, 발사, 둔화 효과 구현, 속도 복원 타이머, 사망 판정, 알림 연결...

2. 수치 등록

3. 레벨업 메뉴 등록

4. 디버그 메뉴 등록

5. 충돌 감지 연결

5개 파일, 100줄+

✅ 이후

설정 파일에 이것만 추가:

{
  이름: 'Frost Bolt',
  타겟: '가장 가까운 적',
  전달: '발사체',
  데미지: 8, 쿨타임: 1.5초,
  효과: [{ 종류: '둔화', 비율: 50%, 지속: 2초 }]
}

1개 파일, 10줄

레벨업 선택지, 디버그 패널, 충돌 처리, 둔화 효과 — 전부 자동으로 연결된다. 무기를 추가하는 사람은 동작 방식을 신경 쓸 필요가 없다.

블록 자체도 늘릴 수 있다

지금까지 보여준 블록(타겟 5종, 전달 5종, 효과 4종, 이벤트 4종)은 처음 만들어 놓은 것일 뿐이다. 나중에 새로운 블록이 필요하면 그 블록 하나만 추가하면 된다.

예: "체인 번개" 전달 방식을 추가하고 싶다면?

전달 블록에 "chain"을 하나 추가하면 끝. 그 순간부터 모든 타겟 방식, 모든 효과, 모든 이벤트와 자유롭게 조합할 수 있다. "가장 가까운 적 + 체인 번개 + 둔화 + 처치 시 회복" 같은 무기도 바로 만들 수 있다.

무기를 새로 만드는 것은 기존 블록을 조합하면 되고, 기존 블록으로 표현할 수 없는 새로운 행동이 필요할 때만 블록을 하나 추가하면 된다. 무기 수가 아무리 늘어나도 블록 수는 천천히 늘어난다.
