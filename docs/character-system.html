캐릭터 & 몬스터 시스템

Vampire Survivors Clone — 플레이어와 몬스터를 같은 구조로 정의하는 데이터 드리븐 시스템

무기를 만들었다. 이제 그 무기를 들 캐릭터를 만들 차례다.

무기 시스템은 완성했다. 블록을 조합하면 무기가 나온다. 그런데 이 무기를 누가 들고 있나? 캐릭터를 정의하고, 무기를 쥐어주는 시스템을 만들어야 한다.

1. 캐릭터를 정의한다

뱀서에는 캐릭터가 수십 명 있다. 전사는 HP가 높고, 마법사는 빠르고, 도적는 투사체가 많다. 각 캐릭터는 스탯 수치로 구분된다.

우리도 이걸 만들고 싶다. 게임 시작 전에 캐릭터를 고르면, 그 캐릭터의 스탯으로 게임이 시작된다. 지금은 HP 100, 속도 200인 캐릭터 1명뿐이고, 이 숫자가 코드에 박혀 있어서 바꿀 수 없다.

캐릭터를 데이터로 정의하면, 숫자만 바꿔서 새 캐릭터를 만들 수 있다.

2. 무기를 쥐어준다

무기 시스템은 이미 만들었다. 채찍, 미사일, 화염탄 같은 무기가 있다. 이 무기를 캐릭터에게 쥐어준다.

캐릭터마다 시작 무기가 다르다. 전사는 채찍으로 시작하고, 마법사는 미사일로 시작한다. 새 캐릭터를 만들면 그 캐릭터에게도 다른 무기를 쥐어줄 수 있다. 적에게도 마찬가지다.

내 캐릭터든 적이든, 같은 무기 시스템에서 무기를 받는다.

3. 스탯이 무기에 전이된다

캐릭터가 "공격력 1.15배"라는 스탯을 가지고 있으면, 그 캐릭터가 든 모든 무기의 데미지가 15% 강해진다. 채찍이든 미사일이든 화염탄이든.

캐릭터의 공격력(might), 쿨다운(cooldown), 범위(area), 투사체 수(amount) 같은 배율 스탯이 무기에 곱해진다. 무기 자체를 건드릴 필요 없이, 캐릭터만 바꾸면 전투가 달라진다.

4. 레벨업하면 성장한다

지금은 레벨이 올라가면 무기를 고르거나 스탯 부스트를 받는다. 하지만 캐릭터 자체의 스탯은 변하지 않는다.

만들고 싶은 건: 캐릭터마다 레벨업할 때 자동으로 올라가는 스탯이 있는 것이다. 전사는 레벨이 오를 때마다 공격력이 2%씩 올라간다. 도적는 20레벨마다 투사체가 1개씩 추가된다. 이게 캐릭터의 정체성이 된다 — 같은 무기를 들어도 후반에 가면 캐릭터마다 전투 스타일이 완전히 달라진다.

5. 플레이어, 몬스터, 보스 — 전부 같은 구조다

위의 1~4 시스템을 플레이어에만 쓸 이유가 없다. 몬스터도 HP가 있고, 속도가 있고, 공격을 한다. 같은 구조다. 이 시스템을 몬스터에게도 그대로 적용한다.

정리하면

캐릭터를 정의하는 시스템이 없다.

스탯이 하드코딩이라 캐릭터를 추가할 수 없고, 스탯이 무기에 전이되지 않아서 캐릭터마다 전투가 달라지지 않는다. 레벨업해도 스탯이 안 올라간다.

몬스터가 무기를 안 들고 있다.

무기 시스템을 만들어 놨는데 플레이어만 쓴다. 몬스터도 무기를 들면 접촉 시 둔화, 원거리 공격, 넉백 같은 것이 전부 데이터로 가능해진다.

플레이어와 몬스터가 같은 개념인데 다른 코드다.

둘 다 HP가 있고, 속도가 있고, 공격한다. 하나로 통합하면 코드도 줄고 기능도 양쪽에 자동으로 적용된다.

어떻게 해결하는가?

캐릭터를 정의하고, 무기를 쥐어주는 시스템을 만든다.

플레이어도, 몬스터도 전부 같은 구조로 정의한다. 스탯이 있고, 행동이 있고, 무기를 든다. 차이는 숫자와 행동 패턴뿐이다.

무기 시스템에서 "무기 = 타겟 + 전달 + 효과 + 이벤트"로 정의한 것처럼, 엔티티도 블록으로 나눈다:

엔티티 = 📊 스탯 + 🎮 행동 + ⚔️ 무기/공격 + 📈 성장

핵심은 스탯이 키-값 구조라는 것이다. Record<string, number>이니까 나중에 armor, luck, growth 같은 스탯을 추가하고 싶으면 키 하나만 넣으면 된다. 코드 수정 없이.

스탯이 무기에 전이되는 방식

캐릭터의 스탯 배율이 무기의 기본 수치에 곱해진다. 이것이 캐릭터마다 같은 무기가 다르게 느껴지는 이유다.

캐릭터를 선택해서 같은 Whip이 어떻게 달라지는지 보세요.

🗡 Whip (기본: 데미지 15, 쿨다운 1500ms, 범위 ×1)
무기 코드를 건드리지 않아도 캐릭터만 바꾸면 전투가 달라진다.

무기는 자기 기본 수치만 가지고 있고, 실제 발동 시점에 캐릭터 스탯을 참조해서 계산한다. 무기 10개가 있어도 스탯 하나만 바꾸면 전부 반영된다.

Block 2: 🎮 행동 (Behavior)

엔티티가 어떻게 움직이고 행동하는지를 정의한다.

🎮 input

키보드/터치 입력으로 이동 (플레이어)

🏃 chase

타겟을 향해 계속 이동

💥 charge

멈췄다가 빠르게 돌진

🏹 ranged

거리를 유지하며 원거리 공격

💣 explode

타겟에 접근하면 자폭

플레이어는 input이고, 기존 몬스터는 전부 chase다. 행동을 바꾸면 같은 스탯의 몬스터도 완전히 다른 적이 된다.

같은 스탯(HP 5, 속도 60)인데 행동만 다른 몬스터를 비교해 보세요.

Block 3: ⚔️ 무기/공격 (Weapons)

이미 만들어둔 무기 시스템을 그대로 쓴다. 핵심은 몬스터도 무기를 가질 수 있다는 것이다.

🧑 플레이어

시작 무기: Whip

레벨업으로 무기 추가 가능

무기 시스템 그대로 사용

👹 원거리 몬스터

무기: 투사체 (데미지 8, 쿨타임 2초)

같은 WeaponDef 구조 사용

접촉 데미지도 "무기"의 일종

접촉 데미지도 무기다.

지금은 접촉 데미지가 별도 로직으로 처리된다. 하지만 이것도 "범위 전체 + 오라 + 데미지 10"인 무기로 표현할 수 있다. 그러면 "접촉 시 둔화"도 그냥 효과를 붙이면 끝이다.

Block 4: 📈 성장 (Growth)

캐릭터마다 레벨업할 때 스탯이 자동으로 올라가는 규칙이다. 뱀파이어 서바이버즈에서 캐릭터 차별화의 핵심이기도 하다.

🗡 전사

매 레벨 might +2%
→ Lv10이면 데미지 1.35배!

🏹 도적

20레벨마다 amount +1
→ Lv40이면 투사체 +2개!

🔮 마법사

매 레벨 cooldown -1%
→ Lv10이면 쿨다운 0.81배!

성장 규칙은 "어떤 스탯을, 몇 레벨마다, 얼마나"라는 3가지 숫자로 정의된다.

{
  growth: [
    { stat: 'might', every: 1, amount: 0.02 },   // 매 레벨 might +2%
  ]
}

캐릭터를 골라서 레벨에 따라 스탯이 어떻게 변하는지 확인해 보세요.

전체 정의 구조

이 모든 것을 합치면, 캐릭터 하나를 정의하는 데 필요한 전부가 하나의 데이터 객체가 된다.

{
  id: 'warrior',
  name: '전사',
  description: '강한 공격력의 기본 캐릭터',

  // 📊 기본 스탯
  stats: {
    maxHp: 120, moveSpeed: 1.0, magnet: 1.0,
    might: 1.15, cooldown: 1.0, area: 1.0, amount: 0,
  },

  // ⚔️ 시작 무기
  startingWeapon: 'whip',

  // 📈 레벨업 성장
  growth: [
    { stat: 'might', every: 1, amount: 0.02 },
  ],
}

몬스터도 같은 구조다. 차이는 행동 패턴과 스탯 값뿐이다.

{
  id: 'charger',
  name: '돌진 몬스터',

  stats: {
    maxHp: 8, moveSpeed: 0.7,
    might: 1.5, cooldown: 1.0, area: 1.0, amount: 0,
  },

  // 🎮 행동 패턴
  behavior: 'charge',

  // ⚔️ 접촉 데미지 (무기로 정의)
  contactDamage: 25,
  contactEffects: [
    { type: 'knockback', force: 300 },
  ],
}

비교: "돌진 몬스터"를 추가한다면?
